Best Practices Code Review How to Protocol Communication Git iOS Open Source Product Review Rails Security Style High level guidelines: Be consistent. Dont rewrite existing code to follow this guide. Dont violate a guideline without a good reason. A reason is good when you can convince a teammate. A note on the language: "Avoid" means dont do it unless you have good reason. "Dont" means theres never a good reason. "Prefer" indicates a better option and its alternative to watch out for. "Use" is a positive instruction.
